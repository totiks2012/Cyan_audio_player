# Cyan & Cy (CGO + libmpv Edition)

**Cyan** и **Cy** — это два минималистичных терминальных аудиоплеера на Go, управляющие встроенным движком `libmpv` через прямой мост CGO. 

> **Историческая справка:** Изначально автор поддался еретическому влиянию и реализовал управление плеером через IPC-сокеты (`/tmp/cyan.sock`), безбожно плодя внешние процессы, задержки и мусор в файловой системе. Но затем автор искренне покаялся и обрёл ... истинный покой в CGO. Теперь всё управление работает прямо в памяти процесса — монолитно, стабильно и в строгом соответствии с канонами философии KISS/Suckless.

Репозиторий предлагает две принципиально разные концепции интерфейса и логики работы с плейлистами. Каждый файл полностью самодостаточен и не имеет внешних тяжелых зависимостей... Ну как... Зависит от небольшой libmpv2

---

### Плеер 1: `cyan` — Эстетика и ручная отрисовка (Bubble Tea)
*Исходный код: `c_main.go`*

* **Интерфейс:** Построен на базе `bubbletea` и `lipgloss`. Полная свобода пиксельной разметки, тонкий кастомный прогресс-бар и неоновый CLI-дизайн для ценителей визуальной эстетики.
* **Логика работы:** Классическая раздельная модель. Плейлист существует независимо от файлового менеджера — треки можно собирать из разных папок в одну общую очередь. Состояние (плейлист, позиция, громкость) сохраняется между перезапусками.

### Плеер 2: `cy` — Сисадминский минимализм (tview/tcell)
*Исходный код: `main.go`*

* **Интерфейс:** Построен на базе виджетов `tview`. Железная стабильность сетки интерфейса. Неубиваем, предсказуем и не «складывается» при любых изменениях размеров окна терминала.
* **Логика работы:** Концепция *«Директория как плейлист»*. Идеально подходит для прослушивания музыкальных альбомов и аудиокниг. Плеер считает текущую папку плейлистом, играет файлы по порядку и автоматически запоминает позицию воспроизведения (секунда в секунду) для каждого трека отдельно.

---

### Сборка и запуск

**Требования для компиляции (разработчику):**
В системе должны быть установлены заголовки `libmpv` (пакет `libmpv-dev` в Debian/Ubuntu или `mpv` в Arch Linux).
а также  build-essential , `sudo apt install build-essential libmpv-dev`

**Требования для работы бинарника (пользователю):**
Для запуска уже скомпилированного плеера тащить за собой тяжелый плеер или пакеты разработки не нужно — достаточно установить в систему легковесную библиотеку `libmpv2` (в Debian/Ubuntu она весит всего около 2–3 мегабайт).

**Сборка красивой версии (`cyan`):**
```
bash
go build -o cyan cyan_main.go
./cyan
```

Сборка утилитарной версии (cy):
Bash
```
go build -o cy cy_main.go
./cy [/путь/к/медиатеке]
```
Оба плеера поддерживают полноценное управление мышью (клик для выбора, скролл списков колесиком) и намертво глушат внутренний логирующий спам от mpv/pipewire, защищая терминал от визуального мусора.

### Плеер 1: `cyan` (на базе Bubble Tea)

* **Навигация и Фокус:**
* `↑` / `↓` — перемещение курсора по списку вверх/вниз.


* `TAB` — переключение фокуса между панелью файлов и панелью плейлиста.


* `ENTER` / `→` — зайти в папку / добавить аудиофайл (или `.m3u` плейлист) в очередь / запустить трек из плейлиста.


* `←` — подняться на одну директорию вверх.




* **Управление воспроизведением:**
* `ПРОБЕЛ` — пауза / возобновление воспроизведения.


* `n` — переключить на следующий трек в плейлисте.


* `,` — перемотка назад на 5 секунд.


* `.` — перемотка вперед на 5 секунд.


* `-` / `_` — уменьшить громкость на 5%.


* `=` / `+` — увеличить громкость на 5%.




* **Плейлист и поиск:**
* `/` — включить режим поиска (фильтрации).


* `ESC` / `ENTER` — выйти из режима поиска (внутри режима).


* `F2` — добавить выбранный файл/директорию в плейлист.


* `F3` — удалить выбранный трек из плейлиста.


* `F5` — полностью очистить текущий плейлист.




* **Выход:**
* `q` / `Ctrl+C` — сохранить состояние и выйти.


* `Q` — сохранить состояние плеера, записать позицию трека (watch-later) и выйти.





---

### Плеер 2: `cy` (на базе tview)

* **Навигация:**
* `↑` / `↓` — перемещение по списку файлов.


* `ENTER` — открыть папку / запустить воспроизведение файла / зайти в `.m3u` плейлист.


* `←` — вернуться на уровень вверх (в родительскую директорию) или выйти из режима просмотра `.m3u`.




* **Управление воспроизведением:**
* `Ctrl+P` — пауза / возобновление воспроизведения.


* `n` / `N` — принудительно переключить на следующий трек в папке.


* `[` — перемотка назад на 5 секунд.


* `]` — перемотка вперед на 5 секунд.


* `-` / `_` — уменьшить громкость на 5%.


* `=` / `+` — увеличить громкость на 5%.




* **Поиск и Сброс:**
* Любой буквенный символ (rune) вне системных команд запускает мгновенную fzf-подобную фильтрацию списка.


* `Ctrl+U` — полностью очистить строку поиска.


* `ESC` — выйти из режима просмотра плейлиста `.m3u` назад в папку или очистить поисковый фильтр.


* `Alt+d` — жесткий сброс: удаляет файл сохраненного состояния трека, историю сессии и обнуляет текущий трек.




* **Системные и Выход:**
* `Ctrl+Z` — приостановить процесс плеера в терминале (SIGTSTP).


* `Ctrl+Q` — остановить поток телеметрии, сохранить текущую позицию воспроизведения и выйти.

# 🌀 CYAN v6.1.0 — Persistent CLI Media Companion

**CYAN** — это минималистичный TUI-плеер (Terminal User Interface), написанный на Go. Он использует **mpv** как мощный бэкенд и IPC-управление для создания надежной рабочей среды.

## 🛠 Почему CYAN?

Проект родился как решение проблемы **персистентности** (сохранения контекста) в CLI-плеерах.

* **Проблема**: В большинстве консольных плееров при выходе теряется всё — позиция поиска, положение в дереве каталогов, текущий трек.
* **Решение**: CYAN сохраняет состояние в реальном времени. При перезапуске вы оказываетесь ровно там, где закончили.
* **Почему не Lua-плагины для mpv?**: Автор изучил существующие скрипты и надстройки, но нашел их либо перегруженными, либо неудобными в навигации. CYAN — это попытка сделать "чистый" инструмент под свои требования к эргономике.

## ⌨️ Управление (Keyboard-Only)

| Клавиша | Действие |
| --- | --- |
| `TAB` | Переключение фокуса (Файлы <-> Плейлист) |
| `↑ / ↓` | Навигация по спискам |
| `ENTER / >` | Войти в папку / Добавить файл в плейлист / Запустить трек |
| `< / ,` | На уровень выше (органический сброс поиска) |
| `/` | **Поиск** (поддерживает Unicode/Кириллицу) |
| `SPACE` | Пауза / Воспроизведение |
| `N` | **Следующий трек** |
| `← / →` | Перемотка ±5 секунд |
| `- / +` | Громкость (шаг 5%) |
| `F2` | Добавить **все** медиафайлы из текущей папки |
| `F3` | **Удалить** выбранный трек из плейлиста |

| `Q` | Выход с сохранением позиции (Watch Later) |
| `ctrl+c` | Быстрый выход |

## 🏗 Ритуал сборки (Deployment Guide)

Для тех, кто впервые сталкивается с Go-средой, приводится полный цикл развертывания.

### 1. Подготовка бэкенда

Убедитесь, что в системе установлен `mpv`:

```bash
# Для Debian/Ubuntu/Mint:
sudo apt install mpv
# Для Arch:
sudo pacman -S mpv

```

### 2. Установка Go (Golang)

Если `go version` не выдает результат, установите среду:

```bash
# Для Linux:
sudo apt install golang  # или скачайте с go.dev/dl

```

Убедитесь, что пути настроены (добавьте в `~/.bashrc` или `~/.zshrc`):

```bash
export GOPATH=$HOME/go
export PATH=$PATH:/usr/local/go/bin:$GOPATH/bin

```

### 3. Сборка CYAN

```bash
# Склонируйте репозиторий
git clone https://github.com/totiks2012/Cyan_audio_player.git
cd Cyan_audio_player

# Инициализация модулей (если собираете впервые)
go mod init cyan
# Установка необходимых библиотек (TUI фреймворк)
go get github.com/charmbracelet/bubbletea
go get github.com/charmbracelet/lipgloss

# Автоматическая очистка и загрузка зависимостей
go mod tidy

# Финальная компиляция
go build -o cyan .

```

### 4. Запуск

**В репо уже есть скомпилированый файл cyan**

```bash
./cyan

```

После первого запуска или вручную можно создать `config.json` в папке проекта:

```json
{
  "theme_color": "#00FFFF",
```
